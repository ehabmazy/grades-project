# 🛠️ دليل الإعداد التفصيلي

## الخطوة 1 — إنشاء مستودع GitHub

1. سجّل دخولك على [github.com](https://github.com)
2. اضغط **New repository** (+ في أعلى اليمين)
3. اختر اسماً مثل: `grades-hamza` أو `school-grades`
4. اجعله **Public** (ضروري لـ GitHub Pages المجاني)
5. **لا** تضع علامة على "Add README" (لأن الملف موجود بالفعل)
6. اضغط **Create repository**

---

## الخطوة 2 — رفع الملفات

### الطريقة السهلة (عبر الموقع):
1. في صفحة المستودع الجديد، اضغط **Add file → Upload files**
2. اسحب جميع الملفات وأفلتها:
   - `index.html`
   - `README.md`
   - `LICENSE`
   - `.gitignore`
   - مجلد `docs/`
3. في خانة "Commit changes" اكتب: `رفع النسخة الأولى`
4. اضغط **Commit changes**

### الطريقة المتقدمة (عبر Git):
```bash
git init
git add .
git commit -m "رفع النسخة الأولى من سجل الدرجات"
git branch -M main
git remote add origin https://github.com/<اسمك>/<المستودع>.git
git push -u origin main
```

---

## الخطوة 3 — تفعيل GitHub Pages

1. اذهب إلى **Settings** في المستودع
2. في القائمة الجانبية اختر **Pages**
3. تحت **Source** اختر:
   - Branch: `main`
   - Folder: `/ (root)`
4. اضغط **Save**
5. انتظر 1-2 دقيقة ثم افتح الرابط المعروض

الرابط سيكون:
```
https://<اسم-المستخدم>.github.io/<اسم-المستودع>/
```

---

## الخطوة 4 — التحقق من النشر

- ستظهر نقطة خضراء ✅ بجانب آخر commit عند اكتمال النشر
- يمكن متابعة حالة النشر من تبويب **Actions**

---

## تحديث التطبيق لاحقاً

عند وجود نسخة جديدة من الملف:

**عبر الموقع:**
1. اذهب إلى ملف `index.html` في المستودع
2. اضغط أيقونة القلم ✏️ (Edit)
3. احذف المحتوى القديم والصق الجديد
4. اضغط **Commit changes**

**عبر Git:**
```bash
# استبدل الملف القديم بالجديد، ثم:
git add index.html
git commit -m "تحديث التطبيق إلى النسخة الجديدة"
git push
```
